# CLI リファレンス

`stratoclave` は、サインイン、API キーの発行、ゲートウェイ経由での Claude 実行、ユーザーとテナントの管理、使用量の確認といった、ユーザー向けおよび管理者向けのすべての操作をまとめた単一の CLI である。本ドキュメントは、すべてのサブコマンド、フラグ、環境変数の権威あるリファレンスである。

タスク指向のウォークスルーは、先に [GETTING_STARTED.md](GETTING_STARTED.md) と [ADMIN_GUIDE.md](ADMIN_GUIDE.md) を読み、オプションの厳密な形状が必要になったときに本ドキュメントへ戻ってくること。

## インストール

現時点では Stratoclave はビルド済みの CLI バイナリを配布していない。ソースからビルドすること。

```bash
git clone https://github.com/littlemex/stratoclave.git
cd stratoclave/cli
cargo build --release
```

バイナリは `target/release/stratoclave` に生成される。`PATH` に配置する。

```bash
# オプション A: 既に PATH に含まれるディレクトリにシンボリックリンクを張る
sudo ln -sf "$PWD/target/release/stratoclave" /usr/local/bin/stratoclave

# オプション B: シェル alias
echo "alias stratoclave='$PWD/target/release/stratoclave'" >> ~/.zshrc   # または ~/.bashrc
source ~/.zshrc

stratoclave --help
```

> **ロードマップ**: macOS (arm64 / x86_64) と Linux (x86_64) 向けに署名済みバイナリを `https://github.com/littlemex/stratoclave` の GitHub Releases ページへ公開するリリースパイプラインは、将来のバージョンで対応予定である。それまではソースからのビルドが公式のパスである。

## コマンドツリー

```
stratoclave <command> [args...]

Commands:
  setup <api_endpoint>        Bootstrap ~/.stratoclave/config.toml from a deployment URL.
  auth
    login                     Cognito email + password (handles first-login challenge).
    sso                       Exchange an AWS SSO / STS session for a Stratoclave token.
    whoami                    Show the current session and credit summary.
    logout                    Clear local tokens.
  claude [-- args...]         Launch claude (Claude Code) through the Stratoclave proxy.
  ui open | url               Open the web console (or print the URL).
  usage
    show                      Your own usage summary and recent history.
  api-key
    create                    Issue a long-lived key (plaintext shown once).
    list                      List your own keys.
    revoke <key_hash>         Revoke one of your keys by SHA-256 hash.
    admin-list                Admin only: list every key in the system.
    admin-revoke <key_hash>   Admin only: revoke any key by SHA-256 hash.
  admin
    user     create | list | show | delete | assign-tenant | set-credit
    tenant   create | list | show | delete | set-owner | members | usage
    usage    show
  team-lead
    tenant   create | list | show | members | usage
```

各階層でフラグレベルのヘルプが必要なときは `stratoclave <command> --help` を実行する。

引数なしで CLI を起動し標準入力がパイプされている場合、CLI はレガシー統合で使われる非対話パイプモードに入る。このモードは未ドキュメントで、新規の用途では依存すべきではない。上記の明示的なサブコマンドを使うこと。

> **重要**: `admin user` や `admin tenant` の複数形は存在しない。`stratoclave admin users list` や `stratoclave admin tenants list` は**存在しない**。常に単数形の名詞 (`admin user list`, `admin tenant list`) を使うこと。

## 設定ファイル

CLI は次のソースから設定を読み込む。優先順位が高い順に並んでいる。

1. 環境変数 ([環境変数](#環境変数) 参照)
2. `~/.stratoclave/config.toml` (または `STRATOCLAVE_CONFIG_DIR` が指すパス)
3. 組み込みのデフォルト値

`config.toml` は `stratoclave setup <api_endpoint>` によって生成される。実稼働のスキーマと一致する正準な形は次の通り。

```toml
# Stratoclave CLI configuration
# Generated by `stratoclave setup` on 2026-04-27T00:00:00Z

[api]
endpoint = "https://d8b03j8erit4k.cloudfront.net"   # your deployment URL

[auth]
auth_method = "cognito"
client_id = "xxxxxxxxxxxxxxxxxxxxxxxxxx"
cognito_domain = "https://stratoclave.auth.us-east-1.amazoncognito.com"
region = "us-east-1"
user_pool_id = "us-east-1_EXAMPLE"

[defaults]
model = "us.anthropic.claude-opus-4-7"

[callback]
host = "127.0.0.1"
port = 18080

[timeouts]
http_total = 10
connection = 5
sse_chunk = 20
auth_callback = 300
```

> **`setup` の出力ラベルに関する注記**: `stratoclave setup` が印字するサマリはモデルを歴史的理由から `cli.default_model` として描画するが、ファイル自体は同じ値を `[defaults] model` 以下に格納している。値は同一で、表示ラベルだけが異なる。

### `STRATOCLAVE_API_ENDPOINT` は実質的に必須である

いくつかのサブコマンド (`auth login`, `admin ...`, `team-lead ...`, `api-key ...`, `usage show`) は、API エンドポイントを `config.toml` の `[api]` セクションではなく、環境変数 `STRATOCLAVE_API_ENDPOINT` から読み込む。これが統一されるまでは、**シェル rc ファイルで `STRATOCLAVE_API_ENDPOINT` をエクスポート**し、すべてのサブコマンドが一貫して動作するようにすること。

```bash
export STRATOCLAVE_API_ENDPOINT="https://d8b03j8erit4k.cloudfront.net"   # your deployment URL
echo 'export STRATOCLAVE_API_ENDPOINT="https://d8b03j8erit4k.cloudfront.net"' >> ~/.zshrc
```

代わりに、レガシーのフラットキーブロックを `config.toml` の先頭に追加してもよい。両方の形式は安全に共存できる。

```toml
# Flat keys read by the subcommands above. Kept for compatibility.
api_endpoint = "https://d8b03j8erit4k.cloudfront.net"
default_model = "us.anthropic.claude-opus-4-7"
```

### トークン

`auth login` または `auth sso` で発行されたトークンは、ファイルモード `0600` で `~/.stratoclave/mvp_tokens.json` に保存される。このファイルを削除することは `stratoclave auth logout` と等価である。

## 環境変数

| 変数 | 用途 | デフォルト |
|----------|---------|---------|
| `STRATOCLAVE_API_ENDPOINT` | API エンドポイントを上書きする。ほとんどのサブコマンドで必須。 | 存在すれば `config.toml` から。 |
| `STRATOCLAVE_DEFAULT_MODEL` | `[defaults] model` を上書きする。 | `claude-opus-4-7`。 |
| `STRATOCLAVE_CONFIG_DIR` | `config.toml` と `mvp_tokens.json` の別置場所。 | `$HOME/.stratoclave`。 |
| `STRATOCLAVE_CLIENT_ID` | Cognito App Client ID を上書きする。 | `config.toml` から。 |
| `STRATOCLAVE_COGNITO_DOMAIN` | Cognito hosted-UI ドメインを上書きする。 | `config.toml` から。 |
| `STRATOCLAVE_ADMIN_UI_URL` | `ui open` が使う URL を上書きする。 | `STRATOCLAVE_API_ENDPOINT` にフォールバック。 |
| `STRATOCLAVE_CALLBACK_PORT` | ローカル OAuth コールバックサーバが使うポート。 | `18080`。 |
| `STRATOCLAVE_REDIRECT_HOST` | ローカル OAuth コールバックサーバが使うホスト。 | `127.0.0.1`。 |
| `AWS_PROFILE` | `--profile` 省略時の `auth sso` のデフォルトプロファイル。 | `default`。 |
| `AWS_REGION` | `--region` 省略時の `auth sso` のデフォルト STS リージョン。 | `us-east-1`。 |
| `ANTHROPIC_BASE_URL` | `stratoclave claude` が `claude` に伝搬する。 | Stratoclave エンドポイント。 |
| `ANTHROPIC_MODEL` | `stratoclave claude` が `claude` に伝搬する。 | `STRATOCLAVE_DEFAULT_MODEL`。 |

## `setup`

デプロイメント URL から CLI 設定をブートストラップする。インストール後の最初のコマンドとして推奨される。

```bash
stratoclave setup <api_endpoint> [--dry-run] [--force]
```

### 引数

| 引数 | 説明 |
|----------|-------------|
| `<api_endpoint>` | Stratoclave デプロイメントのルート URL。たとえば `https://d8b03j8erit4k.cloudfront.net`。`http://` または `https://` で始まる必要がある。`/v1` を含めないこと。CLI が拒否する。 |

### フラグ

| フラグ | 説明 |
|------|-------------|
| `--dry-run` | 生成された `config.toml` をファイルシステムに触れずに標準出力へ出力する。 |
| `-f`, `--force` | 既存の `config.toml` を確認なしで上書きする。元のファイルは `config.toml.bak.<epoch>` として保存される。 |

### 動作

1. URL を検証する (スキーム、ホスト、`/v1` サフィックスなし)。
2. `<api_endpoint>/.well-known/stratoclave-config` を 10 秒のタイムアウトで取得する。
3. `schema_version == "1"` を確認する。
4. `~/.stratoclave/config.toml` が存在する場合、`--force` がなければ確認を求め、`config.toml.bak.<epoch>` にバックアップする。
5. `~/.stratoclave/` が無ければモード `0700` で作成し、モード `0600` で `config.toml` を書き込む。
6. サマリと推奨される次のステップを表示する。

### 例

```bash
stratoclave setup https://d8b03j8erit4k.cloudfront.net --dry-run

stratoclave setup https://d8b03j8erit4k.cloudfront.net
```

期待される出力 (値はデプロイメントに依存する):

```
[INFO] Fetching config from https://d8b03j8erit4k.cloudfront.net/.well-known/stratoclave-config ...

Saved to /home/you/.stratoclave/config.toml
  api_endpoint      = https://d8b03j8erit4k.cloudfront.net
  cognito.domain    = https://stratoclave.auth.us-east-1.amazoncognito.com
  cognito.region    = us-east-1
  cli.default_model = us.anthropic.claude-opus-4-7

Next steps:
  stratoclave auth login --email you@example.com
  # or
  stratoclave auth sso --profile your-sso-profile
```

### よくあるエラー

| エラー | 意味 |
|-------|---------|
| `api_endpoint must start with http:// or https://` | URL にスキームが無い。 |
| `The URL should be the base endpoint, not include /v1` | `/v1` サフィックスを除去する。バックエンドが自動的にルーティングする。 |
| `This Stratoclave deployment does not support /.well-known/stratoclave-config (HTTP 404)` | バックエンドがディスカバリエンドポイントより古い。運用者にアップグレードを依頼する。 |
| `Could not reach <url>` | DNS、ネットワーク、または VPN の問題。管理者と URL を確認する。 |
| `~/.stratoclave/config.toml already exists and stdin is not a TTY` | スクリプト実行時は `--force` を付けて再実行する。 |

## `auth`

認証サブコマンド。すべて `~/.stratoclave/mvp_tokens.json` から読み書きする。

### `auth login`

Cognito のメールアドレスとパスワードでサインインする。初回ログインの場合は同一セッション内で `NEW_PASSWORD_REQUIRED` チャレンジを処理する。

```bash
stratoclave auth login [--email EMAIL] [--password PASSWORD] [--save-password]
```

| フラグ | 説明 |
|------|-------------|
| `--email` | メールアドレス。省略時は対話的に入力を求める。 |
| `--password` | パスワード。省略時はサイレントプロンプトから読み取る。パスワードは端末に入力されるだけで、ブラウザは**開かれない**。 |
| `--save-password` | macOS のみ。パスワードを OS Keychain (サービス名 `stratoclave`) に保存する。オプトイン。 |

成功すると出力の末尾に `[OK] Logged in as <email>. Token saved to ~/.stratoclave/mvp_tokens.json` と表示される。

### `auth sso`

AWS SSO / STS セッションを Stratoclave トークンに交換する。Cognito パスワードは不要。CLI がローカルの AWS クレデンシャルチェーンを使って `sts:GetCallerIdentity` リクエストに署名し、バックエンドが ID を検証する。

```bash
stratoclave auth sso [--profile PROFILE] [--region REGION]
```

| フラグ | 説明 |
|------|-------------|
| `--profile` | AWS プロファイル名。デフォルトは `AWS_PROFILE` または `default`。 |
| `--region` | STS リージョン。デフォルトは `AWS_REGION` または `us-east-1`。 |

バックエンド側の前提条件:

- 自分の AWS アカウント ID が信頼された ID ソースとして登録されている。
- 自分の IAM ロール ARN がデプロイメントの許可ロールパターンに一致する。
- EC2 インスタンスプロファイルから署名していない (デフォルトで拒否される)。

よくある SSO 拒否パターンは [GETTING_STARTED.md](GETTING_STARTED.md#sign-in) に一覧がある。

### `auth whoami`

現在のセッションとクレジットサマリを表示する。出力は 7 行である。

```
email: you@example.com
user_id: a4f824f8-b041-703d-3ec8-f15588b9c969
org_id: default-org
roles: user
total_credit: 1000000
credit_used: 42318
remaining_credit: 957682
```

`roles` はカンマ区切りのリスト (角括弧なし) として描画されるので、admin でもある team lead は `roles: admin,team_lead` と表示される。ログインしていない、またはトークンが失効している場合は非ゼロで終了する。

### `auth logout`

`~/.stratoclave/mvp_tokens.json` を削除する。Cognito セッションはサーバー側では取り消されない。強制失効が必要なら管理者に `AdminUserGlobalSignOut` の実行を依頼する。

```bash
stratoclave auth logout
# [OK] Local tokens cleared
```

## `claude`

`claude` バイナリ ([Claude Code](https://docs.claude.com/en/docs/claude-code/overview) から) を Stratoclave の配線事前済みでサブプロセスとして起動する。

```bash
stratoclave claude [--model MODEL_ID] -- [claude-args...]
```

| フラグ | 説明 |
|------|-------------|
| `--model` | 本呼び出し用に `ANTHROPIC_MODEL` を上書きする。 |
| `-- <args>` | `--` 以降はすべて `claude` バイナリへそのまま渡される。 |

サブプロセスが見る環境変数は以下の通り。

| 環境変数 | 値 |
|----------------------|-------|
| `ANTHROPIC_BASE_URL` | 設定されている Stratoclave エンドポイント。 |
| `ANTHROPIC_API_KEY` | `mvp_tokens.json` の Cognito アクセストークン。 |
| `ANTHROPIC_MODEL` | `--model` が優先、次に `[defaults] model`、最後に `claude-opus-4-7`。 |

Stratoclave は `CLAUDE_CODE_USE_BEDROCK` と `AWS_REGION` の環境変数を unset するため、Claude Code が誤ってプロキシをバイパスすることはない。

### 例

```bash
stratoclave claude -- "Hello, who are you?"

stratoclave claude --model claude-haiku-4-5 -- "Summarise README.md"

stratoclave claude -- --print "List files"
```

### サポートされるモデル ID

| CLI の値 | Bedrock 推論プロファイル |
|-----------|---------------------------|
| `claude-opus-4-7` | `us.anthropic.claude-opus-4-7` |
| `claude-opus-4-6` | `us.anthropic.claude-opus-4-6-v1` |
| `claude-opus-4-5` | `us.anthropic.claude-opus-4-5-20251101-v1:0` |
| `claude-sonnet-4-6` | `us.anthropic.claude-sonnet-4-6` |
| `claude-sonnet-4-5` | `us.anthropic.claude-sonnet-4-5-20250929-v1:0` |
| `claude-haiku-4-5` | `us.anthropic.claude-haiku-4-5-20251001-v1:0` |

実効的なリストはデプロイメントで許可されている Bedrock 推論プロファイルに依存する。期待するモデルが無ければ管理者に確認すること。

## `ui`

### `ui open`

既定のブラウザで Stratoclave の Web コンソールを開く。CLI が URL に `?token=<access_token>` を付与するため、コンソールは認証済みの状態で表示される。

```bash
stratoclave ui open
```

ログインしていない、またはトークンが失効している場合は明確なメッセージで失敗する。

### `ui url`

URL (トークン付き) を標準出力に表示する。SSH 越しやサンドボックス内で URL を別の場所へコピーしたいときに便利である。

```bash
stratoclave ui url
# https://d8b03j8erit4k.cloudfront.net?token=eyJ...
```

出力はシークレットとして扱うこと。git にコミットしたりチケットに貼り付けたりしないこと。

## `api-key`

長寿命 API キー (`sk-stratoclave-...`) は、Claude Desktop Cowork、CI スクリプト、カスタム SDK 統合のような非対話型クライアントを認証するために使える。各ユーザーはデプロイメントで設定された上限 (デフォルト: アクティブ 5 個) まで保持できる。

### `api-key create`

新しいキーを作成する。プレーンテキスト値は**一度だけ**表示される。

```bash
stratoclave api-key create [--name NAME] [--scope SCOPE]... [--expires-days DAYS]
```

| フラグ | デフォルト | 説明 |
|------|---------|-------------|
| `--name` | 空 | 人間可読なラベル。最大 64 文字。たとえば `"cowork on macbook"`。 |
| `--scope` | `messages:send`, `usage:read-self` | キーに埋め込むパーミッション。複数指定可。現在のロールのサブセットでなければならない。 |
| `--expires-days` | `30` | 有効日数 (`1` から `3650`)。`0` を指定すると失効しないキー (非推奨)。 |

サンプル出力:

```
  ==========================================================
   API Key created (save this now - it won't be shown again)
  ==========================================================

   sk-stratoclave-4f8c9b2a1d7e6c0f3a5b8c9d2e1f4a7b

   key_id     : sk-stratoclave-4f8c...4a7b
   scopes     : messages:send, usage:read-self
   expires_at : 2026-05-27T00:00:00Z

  Paste this value into `Gateway API key` (Claude Desktop cowork)
  or use it directly as `Authorization: Bearer <key>`.
```

`key_id` は `sk-stratoclave-XXXX...YYYY` の形式 (本体の先頭 4 文字と末尾 4 文字、真ん中はマスク) の表示用識別子である。ログに残しても安全だが、プレーンテキストは安全ではない。

### `api-key list`

自分のキーを一覧表示する。プレーンテキストは二度と表示されない。

```bash
stratoclave api-key list [--include-revoked]
```

サンプル出力:

```
active: 2 / 5
  sk-stratoclave-4f8c...4a7b
    name: cowork on macbook
    scopes: messages:send,usage:read-self
    expires_at: 2026-05-27T00:00:00Z
    last_used_at: 2026-04-27T10:15:02Z
  sk-stratoclave-1a2b...9c0d [REVOKED]
    name: ci-pipeline
    scopes: messages:send
    expires_at: no-expiration
    last_used_at: never
```

この出力からの取り消しに関する注意点は [既知の制限](#既知の制限) を参照。

### `api-key revoke`

自分のキーの 1 つを取り消す。即時に有効化され、次のリクエストで当該キーを使うと `401 Unauthorized` が返る。

```bash
stratoclave api-key revoke <key_hash>
```

引数はプレーンテキストの SHA-256 hex ダイジェスト (DynamoDB のパーティションキー) であり、`api-key list` が表示する `key_id` では**ない**。このギャップが埋まるまでの推奨ワークフローは [既知の制限](#既知の制限) を参照。

### `api-key admin-list` と `admin-revoke`

管理者専用: 任意ユーザーが所有するキーを検査・取り消しする。

```bash
stratoclave api-key admin-list [--include-revoked]
stratoclave api-key admin-revoke <key_hash>
```

`admin-list` の出力は同じ行に `owner=<user_id>` とキーの `name` も追加で出力する。

## `admin`

`admin` ロールが必要である。他のロールは HTTP 403 を受け取る。

### `admin user create`

新規ユーザーをプロビジョニングする。デフォルトでは `default-org` テナントにテナントの `default_credit` を持つ `user` ロールのアカウントを作成する。

```bash
stratoclave admin user create --email user@example.com \
  [--role user|team_lead|admin] \
  [--tenant TENANT_ID] \
  [--total-credit N]
```

> **重要**: デフォルトではレスポンスに `temporary_password` フィールドは**含まれない**。値は `null` である。これは意図的な硬化措置である。管理者は初回ログイン用クレデンシャルをアウトオブバンドで発行しなければならない。推奨ワークフローは次の通り。
>
> ```bash
> # 1. Stratoclave でユーザーを作成する。
> stratoclave admin user create --email newuser@example.com --role user --tenant default-org
>
> # 2. Cognito で一時パスワードを設定し、安全なチャネルでユーザーに渡す。
> aws cognito-idp admin-set-user-password \
>   --user-pool-id us-east-1_EXAMPLE \
>   --username newuser@example.com \
>   --password 'TempPassword!23' \
>   --no-permanent \
>   --region us-east-1
> ```
>
> `--no-permanent` はユーザーを `FORCE_CHANGE_PASSWORD` 状態にするため、最初の `stratoclave auth login` で新しいパスワードを選ぶことが求められる。
>
> レガシーツール向けに create レスポンスで `temporary_password` を返す必要がある場合は、デプロイ前に ECS タスク定義の環境変数 `EXPOSE_TEMPORARY_PASSWORD=true` を設定する。本番環境では非推奨である。

### `admin user list`

```bash
stratoclave admin user list [--role R] [--tenant T] [--limit N]
```

`--limit` のデフォルトは `50` である。

### `admin user show`

```bash
stratoclave admin user show <user_id>
```

### `admin user delete`

ユーザーを Cognito から削除し、主 `Users` 行を削除する。過去の `UserTenants` エントリはアーカイブ (削除されない) されるので、使用量ログの帰属は維持される。

```bash
stratoclave admin user delete <user_id>
```

ガードレール: 自分自身の削除は HTTP `409` で拒否される。最後の `admin` の削除も HTTP `409` で拒否される。

### `admin user assign-tenant`

ユーザーを別のテナントへ移動する。ロールクレームが変わるため、ユーザー側には強制再ログインが発生する。

```bash
stratoclave admin user assign-tenant <user_id> \
  --tenant <new_tenant_id> \
  [--new-role user|team_lead] \
  [--total-credit N]
```

### `admin user set-credit`

ユーザーのクレジット予算を上書きする。

```bash
stratoclave admin user set-credit <user_id> --total N [--reset-used]
```

`--reset-used` は `credit_used` をゼロにするので、たとえば新しい請求期間の開始時にユーザーが新鮮な状態で始められる。

### `admin tenant create`

テナントを作成する。`--team-lead` (Cognito sub) と `--team-lead-email` は最大 1 つだけ指定できる。両方省略すると所有権はセンチネル文字列 `admin-owned` になる。

```bash
stratoclave admin tenant create --name "Team A" \
  [--team-lead <user_id> | --team-lead-email lead@example.com] \
  [--default-credit N]
```

### `admin tenant list`

```bash
stratoclave admin tenant list [--limit N]
```

### `admin tenant show`

```bash
stratoclave admin tenant show <tenant_id>
```

### `admin tenant delete`

テナントをアーカイブする (`status=archived` をセット)。組み込みの `default-org` テナントはアーカイブできない。

```bash
stratoclave admin tenant delete <tenant_id>
```

### `admin tenant set-owner`

テナントの team lead を再割り当てする。監査ログはアクターを記録する。

```bash
stratoclave admin tenant set-owner <tenant_id> \
  [--team-lead <user_id> | --team-lead-email lead@example.com]
```

### `admin tenant members`

`user_id` を含め、テナントのすべてのメンバーを一覧表示する。管理者はデバッグ目的で生の識別子を見られる。

```bash
stratoclave admin tenant members <tenant_id>
```

### `admin tenant usage`

単一テナントの使用量を集計する。

```bash
stratoclave admin tenant usage <tenant_id> [--since-days N]
```

### `admin usage show`

デプロイメント全体の生の使用量ログをクエリする。

```bash
stratoclave admin usage show \
  [--tenant T] \
  [--user U] \
  [--since 2026-04-01T00:00:00Z] \
  [--until 2026-04-30T23:59:59Z] \
  [--limit N]
```

バックエンドはインデックスを `tenant_id > user_id > フルスキャン` の順で選ぶ。スキャンを避けるため、可能な限り `--tenant` や `--user` を渡すこと。

## `team-lead`

`team_lead` ロールが必要である (admin も使えるが、所有するテナントに対してのみ)。所有者でない場合は、列挙を防ぐために未知のテナントに対して HTTP 404 が返される。

### `team-lead tenant create`

呼び出し元が所有するテナントを作成する。`team_lead_user_id` はバックエンドによって強制されるため、他のオーナーを偽装することはできない。

```bash
stratoclave team-lead tenant create --name "My Team" [--default-credit N]
```

### `team-lead tenant list`

自分が所有するテナントのみを一覧表示する。他のテナントは不可視である (`403 Forbidden` ではなく、そもそも返されない)。

```bash
stratoclave team-lead tenant list
```

### `team-lead tenant show`

```bash
stratoclave team-lead tenant show <tenant_id>
```

### `team-lead tenant members`

自分が所有するテナントのメンバーを一覧表示する。`admin tenant members` と違い、`user_id` は返されない。メールアドレスとクレジットフィールドのみである。

```bash
stratoclave team-lead tenant members <tenant_id>
```

### `team-lead tenant usage`

```bash
stratoclave team-lead tenant usage <tenant_id> [--since-days N]
```

| コード | 意味 |
|------|---------|
| `0` | 成功。 |
| `1` | CLI が報告した何らかのエラー (ネットワーク、検証、サーバーエラー、パーミッション拒否、見つからないなど)。stderr のメッセージがどれかを説明する。 |

## 既知の制限

以下は現時点の CLI で追跡されている既知のギャップである。各項目は後続リリースで解消される予定である。それまでは、ドキュメントされた回避策を使うこと。

### `api-key revoke` の引数は `key_id` ではなく SHA-256 ハッシュである

`api-key revoke` はプレーンテキストの `key_hash` (SHA-256 hex) を必要とするが、`api-key list` は現在、`key_hash` を出力に含めていない。代わりにマスク済みの `key_id` (`sk-stratoclave-XXXX...YYYY`) が表示される。

リスト出力が拡充されるまでは、次のいずれかで取り消すこと。

- **Web UI**: 自分自身のキーなら *Account -> API keys -> Revoke*、任意ユーザーのキーなら *Admin -> API keys*。
- **HTTP API を直接叩く**: `DELETE /api/mvp/me/api-keys/{key_hash}` (管理者は `GET /api/mvp/admin/api-keys` でハッシュを取得できる)。

### `admin user create` はデフォルトでは一時パスワードを返さない

[`admin user create`](#admin-user-create) を参照。ユーザーを作成した後、`aws cognito-idp admin-set-user-password --no-permanent` で初回ログイン用パスワードをプロビジョニングすること。レガシー動作はバックエンド側で `EXPOSE_TEMPORARY_PASSWORD=true` を設定することで再有効化できる。

### `admin trusted-accounts` に CLI サブコマンドがない

`auth sso` が使う信頼されたアカウントの許可リストは、Web UI から管理するか、HTTP API `POST /api/mvp/admin/trusted-accounts` を直接呼び出して管理する。`stratoclave admin trusted-account ...` 系のサブコマンドは計画中である。

### SSO nonce リプレイウィンドウ

バックエンドは署名済み `GetCallerIdentity` リクエストの `X-Amz-Date` が自身のウォールクロックの ±5 分以内であることを検証するが、まだ署名ごとの nonce を記録していない。将来のリリースでは 5 分 TTL の STS nonce テーブルを追加してリプレイウィンドウを無くす予定である。

## 典型的なワークフロー

### 新規ユーザーのオンボーディング (メールアドレス + パスワード)

```bash
# 管理者: Stratoclave のレコードを作成し、Cognito で一時パスワードを渡す。
# なぜ 2 ステップなのかは `admin user create` を参照。
stratoclave admin user create --email alice@example.com --role user --tenant default-org

aws cognito-idp admin-set-user-password \
  --user-pool-id us-east-1_EXAMPLE \
  --username alice@example.com \
  --password 'Temp!PickAStrongOne23' \
  --no-permanent \
  --region us-east-1
# -> 一時パスワードを安全なチャネルで Alice に届ける。

# Alice
stratoclave setup https://d8b03j8erit4k.cloudfront.net
export STRATOCLAVE_API_ENDPOINT="https://d8b03j8erit4k.cloudfront.net"
stratoclave auth login --email alice@example.com
stratoclave ui open
```

### SSO でチーム全体をオンボーディングする

```bash
# 管理者 (一度だけ): Admin UI または POST /api/mvp/admin/trusted-accounts で AWS アカウントを
# 信頼された ID ソースとして登録する。

# 各チームメンバー
aws sso login --profile team
stratoclave setup https://d8b03j8erit4k.cloudfront.net
stratoclave auth sso --profile team
stratoclave ui open
```

### 日常的な Claude Code のラッピング

```bash
stratoclave auth login --email you@example.com    # または `auth sso`
stratoclave claude -- "Refactor this Python function"

# 401 が返ったらセッションを更新する。
stratoclave auth login --email you@example.com
```

```bash
stratoclave api-key create \
  --name "cowork on macbook" \
  --scope messages:send \
  --scope usage:read-self \
  --expires-days 90
# 出力された sk-stratoclave-... を Claude Desktop の Gateway API key フィールドに貼り付ける。
```
